Final Architecture (Clean Separation)

PacWorldImporter → Responsible for loading PacAi export, parsing both world.pacdata.json and visual_manifest.json, and coordinating initialization.
PacSimulation (existing) → Owns the deterministic simulation (entities, ConflictSim, behaviors, etc.).
PacRenderer → Purely visual — only deals with rendering proxies, materials, lighting, fog, camera, etc.

    : m_simulation(simulation), m_renderer(renderer) {}

bool PacWorldImporter::Import(const std::string& exportFolderPath) {
    if (!fs::exists(exportFolderPath)) {
        std::cerr << "[PacWorldImporter] Export folder not found: " << exportFolderPath << std::endl;
        return false;
    }

    std::cout << "[PacWorldImporter] Starting import from: " << exportFolderPath << std::endl;

    // 1. Ensure asset directories exist
    CreateAssetDirectories(exportFolderPath);
    CreatePlaceholderAssets(exportFolderPath);

    // 2. Load Visual Manifest (for rendering)
    VisualManifest manifest;
    std::string manifestPath = exportFolderPath + "/visual_manifest.json";
    bool hasVisual = VisualManifestLoader::Load(manifestPath, manifest);

    if (hasVisual && m_renderer) {
        m_renderer->ApplyVisualManifest(manifest);
    }

    // 3. Load PacData world (for simulation)
    PacDataWorld world;
    std::string pacdataPath = exportFolderPath + "/world.pacdata.json";
    bool hasPacData = PacDataLoader::Load(pacdataPath, world);

    if (hasPacData && m_simulation) {
        m_simulation->LoadWorld(world);        // This is where your simulation core takes over
    } else if (!hasPacData) {
        std::cerr << "[PacWorldImporter] Warning: Failed to load PacData world." << std::endl;
    }

    // 4. Bridge visual entities to simulation entities if needed
    if (hasVisual && hasPacData) {
        ApplyVisualToSimulation(manifest, world);
    }

    std::cout << "[PacWorldImporter] Import completed. "
              << "PacData: " << (hasPacData ? "OK" : "Failed") << ", "
              << "Visual: " << (hasVisual ? "OK" : "Failed") << std::endl;

    return hasPacData || hasVisual;  // At least one should succeed
}

bool PacWorldImporter::CreateAssetDirectories(const std::string& exportFolderPath) {
    fs::path assetsDir = fs::path(exportFolderPath) / "assets";
    fs::path modelsDir = assetsDir / "models";
    fs::path terrainDir = assetsDir / "models" / "terrain";

    try {
        fs::create_directories(modelsDir);
        fs::create_directories(terrainDir);
        return true;
    } catch (...) {
        std::cerr << "[PacWorldImporter] Failed to create asset directories." << std::endl;
        return false;
    }
}

void PacRenderer::UpdateSimulationState(const PacDataWorld& worldDelta) {

        m_scene->UpdateProxyTransform(ent.id, buildTransform(ent.transform));

Summary of Changes

PacWorldImporter is now the single responsible class for importing a PacAi export.
PacRenderer is now purely visual — no more PacData loading inside it.
Clear ownership: Importer → Simulation + Renderer.
Much easier to test and maintain.
