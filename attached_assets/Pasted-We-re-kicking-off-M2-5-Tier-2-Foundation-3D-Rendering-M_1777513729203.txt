Step 1: High-Level Architecture for M2.5 (Solo-Dev Friendly)
We'll keep the strict separation:

Simulation Core (Tier 1) — unchanged, remains the source of truth.
Render Layer — new C++ module that consumes simulation state deltas.
Editor (TS) — orchestrates everything and provides the UI.

Step 2: Tech Choices (2026 Recommendations)
glTF Loader:

Primary recommendation: fastgltf (modern C++17, very fast, SIMD-accelerated, good usability).
Fallback / simpler option: tinygltf (header-only, battle-tested, used in Sascha Willems' Vulkan glTF-PBR example).

Rendering Backend:

Vulkan (via Vulkan-HPP for cleaner C++ API).
Start with Forward+ or simple clustered forward renderer — easier to maintain as a solo dev than full deferred.
Reference implementations:
Sascha Willems' Vulkan-glTF-PBR — excellent starting point for glTF + PBR.
vkguide.dev modern Vulkan tutorials (updated for 2026).


Atmospherics (MVP):

Directional light + simple IBL (image-based lighting) using a HDR sky cubemap.
Basic volumetric fog or height-based fog.
Physical sky model (simple procedural or pre-baked).
